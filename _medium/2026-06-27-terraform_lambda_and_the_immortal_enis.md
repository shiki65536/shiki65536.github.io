This week’s infrastructure teardown perfectly demonstrated what happens when automated IaC meets cloud voodoo.”I fully expected `terraform destroy -auto-approve` to cleanly wrap things up, but instead, the terminal got trapped in a classic endless “Still destroying…” loop while trying to delete a Security Group (SG).The Crime SceneThe AWS API refused to delete the Security Group because it was still attached to several ENIs sitting stubbornly in an `in-use` state.This brings us to one of the most notorious pain points when integrating AWS Lambda VPC networking with Terraform: Lambda VPC ENI asynchronous cleanup.Root CauseLambda VPC networking relies on underlying Hyperplane ENIs to share connections across execution environments. The catch: when you delete a Lambda function, AWS doesn’t release these ENIs immediately, deferring the cleanup to an asynchronous background process.This “sweeper” mechanism is a complete black box, with cleanup times ranging from 15 minutes to over an hour. Terraform is left powerless here; as long as the AWS API reports the ENIs as in-use, Terraform will trigger a DependencyViolation error, stalling your pipeline in the purgatory of &quot;eventual consistency.&quot;The FixDon’t gamble with AWS timing, and don’t just stare at the terminal. When facing this orphaned ENI deadlock, the most pragmatic approach is a direct surgical intervention:1. Untrack from Terraform: Break the spell by telling Terraform to stop managing the stuck SG.# Remove the SG from the local stateterraform state rm aws_security_group.lambda_sg2. Manual Exorcism (ClickOps):Put IaC pride aside, open the AWS Console, manually locate those in-use ENIs, forcefully delete them, and then finally terminate the SG.TakeawayIn the ruthless world of the cloud, sometimes the most efficient automation is knowing exactly when to switch back to manual mode. Rather than waiting an hour for a miracle, it’s better to admit Terraform’s limits and spend three minutes clicking through the Console to finish the job.Reference[AWS Docs] Giving Lambda functions access to resources in an Amazon VPC[GitHub Issue] DependencyViolation when deleting security group associated to lambda and vpc_endpon
