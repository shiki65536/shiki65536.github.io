This week I revived an old university assignment: an object detection photo album.The idea was simple: Upload a photo → detect objects → auto generate tags → search images by tags or by another image.Originally, the main blocker was the YOLO model weights. The trained weights were too large to comfortably fit into a normal Lambda deployment package, and wiring them through S3 was awkward. Back then, I solved it with a Lambda Layer.Deployment ArchitectureThis time, I redeployed the whole thing properly on AWS:CloudFront + S3 for the frontendAPI Gateway for the API layerLambda for backend logicCognito for authenticationDynamoDB for image metadata and tagsS3 for image storageECR + Lambda container image for object detectionThe best upgrade was moving object detection into an ECR-backed Lambda container image. Compared with fighting Lambda package size and layers, this felt almost civilised. Almost.Cold start on the detection Lambda is about 10 seconds. After that it stays warm. Total AWS cost for development and testing: approximately $0. Free tier handles everything comfortably at this scale, which was the whole point.Technical Challenges and ResolutionsThe rest of the deployment reminded me that cloud engineering is mostly debugging invisible glue:CORS was failing because the frontend sent custom headers like `Authorization`, `userID`, and `x-requested-with`. Turns out API Gateway HTTP API and REST API handle CORS differently, and mixing the two means two configs to maintain. Don’t do that.Cognito Google OAuth needed the correct callback URLs, logout URLs, Google redirect URI, and frontend token parsing.S3 no longer accepts the old `public-read` ACL pattern by default, so I removed object-level ACL usage and adjusted the bucket policy for the demo image bucket.The API Lambda was timing out while waiting for object detection, so I increased the API timeout and moved the model into a Lambda container image built from ECR.SummaryA user can log in with email/password or Google, upload an image, wait for YOLO object detection, edit tags, search by tag, and find similar images by uploading another reference image.👉 Link to AI Object Detection Album: SpottedThe funny part is that the AI model was not the hardest part. The hard part was making all the AWS pieces agree on: “yes, this request is allowed, this token is valid, this file exists, this header is acceptable, this cache is not from yesterday, and this Lambda has enough time to breathe.”The model detects objects. The cloud detects your patience.
